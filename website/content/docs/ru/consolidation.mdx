# Консолидация

Консолидация — процесс очистки базы от дубликатов, устаревших и низкокачественных записей. Выполняется в три шага с контролем на каждом этапе.

## Трехшаговый процесс

### 1. Preview — поиск кандидатов

Находит группы дубликатов в два прохода, а также записи, подходящие под фильтры `stale_days` и `min_score`. Не вызывает LLM, не изменяет данные.

- **Точный проход** группирует записи с байт-идентичными триплетами context/action/result (`match_type: "exact"`).
- **FTS-проход** пробует context каждой оставшейся записи по полнотекстовому индексу BM25 и группирует лучшие совпадения (`match_type: "fts"`).

```bash
engram consolidate preview --stale-days 30 --min-score 0.3
```

```json
// MCP
memory_consolidate_preview({ "stale_days": 30, "min_score": 0.3 })
```

Каждая группа дубликатов содержит:

| Поле | Значение |
|------|----------|
| `primary_id` | Детерминированный представитель группы (exact: лексикографически наименьший id; FTS: запись-проба). Не обязательно survivor при merge — analyze выбирает survivor независимо. |
| `duplicate_ids` | Остальные участники группы. |
| `similarity` | Шкала зависит от `match_type`: exact-группы несут сентинел `1.0`; FTS-группы — средний &#124;bm25&#124; rank участников группы (неотрицательный, неограниченный сверху — не cosine). |
| `match_type` | `"exact"` или `"fts"`. |

Настройка `consolidation.fts_similarity_floor` (по умолчанию `0.0` = выключено) отбрасывает FTS-группы, чья similarity ниже порога, до этапа analyze. Exact-группы никогда не фильтруются.

### 2. Analyze — LLM-анализ

Передает кандидатов в LLM для анализа. Возвращает рекомендации с обоснованием: merge, keep, delete, archive.

```bash
engram consolidate analyze
```

```json
// MCP
memory_consolidate({ "stale_days": 30 })
```

Сбои LLM не прерывают batch: каждый неудачный участник или незагружаемая группа добавляет запись в массив `errors` ответа, не порождает рекомендацию и не входит в `analyzed_count`. Без настроенного LLM survivor выбирает детерминированная эвристика (выше score, затем usage, затем более раннее создание).

### 3. Apply — применение

Применяет рекомендации: объединяет, удаляет или архивирует записи. `min_confidence` (по умолчанию `0.0` = применять все) пропускает рекомендации с confidence ниже порога. Массив `errors` ответа объединяет ошибки анализа (с префиксом `analyze ...`) и ошибки применения.

```bash
engram consolidate apply --min-confidence 0.7
```

```json
// MCP
memory_consolidate_apply({ "min_confidence": 0.7 })
```

## Операции

### Merge

Объединяет две или более похожих записей в одну. Новая запись наследует лучшие поля из исходных. Исходные записи получают `superseded_by` с ID новой записи.

### Delete

Полностью удаляет запись из базы данных.

### Archive

Помечает запись как замененную через `superseded_by`. Запись остается в базе, но исключается из поиска.

## Дедупликация при записи

Помимо консолидации, engram выполняет дедупликацию при каждом вызове `memory_store`. Если cosine similarity новой записи с существующей превышает 0.95, существующая запись обновляется вместо создания дубликата.

## Параметры

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `stale_days` | 90 (из конфигурации) | Записи старше этого порога считаются устаревшими |
| `min_score` | 0.3 (из конфигурации) | Записи с score ниже порога — кандидаты на удаление |
| `fts_similarity_floor` | 0.0 (из конфигурации) | Минимальный средний &#124;bm25&#124; rank FTS-группы дубликатов в preview; 0.0 выключает фильтр, exact-группы никогда не фильтруются |

Параметры настраиваются в `engram.toml`:

```toml
[consolidation]
stale_days = 90
min_score = 0.3
fts_similarity_floor = 0.0
```

## Insights

В рамках консолидации и анализа через trainer генерируются insights — производные знания, извлеченные из паттернов в записях. Insights сохраняются как отдельные записи с `memory_type = "insight"` и участвуют в кросс-проектном поиске.

Управление insights:

```bash
engram train list              # список insights
engram train generate          # генерация новых insights
engram train delete --id UUID  # удаление insight
```

## Аудит

Все операции консолидации записываются в таблицу `consolidation_log` с указанием действия и источника (auto/user).
