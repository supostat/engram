# CLI команды

Engram CLI работает автономно с SQLite и HNSW-индексами на диске. Запущенный сервер не требуется.

Глобальные флаги:

- `--config <path>` — путь к файлу конфигурации
- `--format <json|text|jsonl>` — формат вывода (по умолчанию `json`)

## engram init

Создаёт директорию `.engram/` в текущем каталоге проекта (с `engram.db`, минимальным закомментированным project-local `engram.toml`, упакованным `AGENT.md`, `.mcp.json` и записью `.engram/` в `.gitignore`). При первом запуске также пишет дефолтный `~/.engram/engram.toml`, если его нет, и выводит инструкции для подключения MCP-сервера.

```bash
engram init
```

Эта команда обрабатывается до загрузки конфигурации и не требует существующего `engram.toml`.

## engram server

Запускает Unix-сокет сервер для приема JSON-запросов от MCP-сервера.

```bash
engram server
```

Путь к сокету разрешается в порядке: переменная `ENGRAM_SOCKET_PATH` → `<project>/.engram/engram.sock` (если найден маркер `.engram/` при подъёме вверх от cwd, как `.git`) → `server.socket_path` из `engram.toml` (legacy fallback). База данных разрешается аналогично: `ENGRAM_DB_PATH` → `<project>/.engram/engram.db`. Если per-project БД отсутствует, но существует legacy `~/.engram/engram.db`, запуск прерывается с подсказкой запустить `engram migrate`. Фоновая переиндексация выполняется с интервалом `reindex_interval_secs`.

## engram migrate

Импортирует записи из legacy глобальной базы (`~/.engram/engram.db`) в `<project>/.engram/engram.db` текущего проекта. По умолчанию фильтрует строки, у которых поле `project` точно совпадает с basename текущей директории; строки с NULL `project` пропускаются, если не передан `--all`.

```bash
cd /path/to/project
engram migrate --dry-run   # только предпросмотр — ничего не пишется
engram migrate             # импорт строк, чей project совпадает с текущим
engram migrate --all       # также импортирует NULL-project / несовпадающие строки
```

| Флаг | По умолчанию | Описание |
|------|-------------|----------|
| `--all` | выкл | Включить строки, у которых `project` равен NULL или отличается от basename cwd |
| `--dry-run` | выкл | Сообщить, что было бы импортировано, без записи в целевую БД |

## engram reembed

Пересчитывает эмбеддинги для каждой записи в БД проекта с использованием текущих `embedding.provider` и `embedding.model`. Заменяет векторы в HNSW (delete + insert на каждую запись), записывает свежие BLOB'ы в SQLite с `indexed=true`, очищает in-memory кеш в начале и записывает активную модель в `schema_meta.embedding_model` только когда все записи обработаны успешно.

```bash
engram reembed             # one-shot пересчёт с активным провайдером
engram reembed --force     # зарезервировано под будущие safety-пороги (no-op сейчас)
```

Используй после смены `embedding.model` в `engram.toml`. Startup-страж, добавленный в 0.3, не пустит демон стартовать с `[6020] EmbeddingModelMismatch`, пока `engram reembed` не завершится чисто.

Per-memory ошибка (провайдер недоступен, HNSW сбой) сбрасывает `indexed=0` для соответствующей записи — она ретраится в следующем запуске. Run возвращает `failed > 0` в JSON-сводке и **не** обновляет `schema_meta`. Ошибки записи в БД прерывают run целиком и пропагируются. Команда работает по CLI-пути и намеренно обходит startup-страж — иначе пользователь не смог бы починить mismatch.

| Флаг | По умолчанию | Описание |
|------|-------------|----------|
| `--force` | выкл | Зарезервировано под будущие safety-пороги (no-op в 0.3.0) |

## engram store

Сохраняет новую запись в памяти.

```bash
engram store \
  --context "Модуль auth стал монолитным, 800 строк" \
  --action "Разделил на middleware + handler + validator" \
  --result "Каждый файл < 200 строк, тесты изолированы"
```

| Флаг | Обязательный | Описание |
|------|-------------|----------|
| `--context` | да | Ситуация или контекст |
| `--action` | да | Предпринятое действие |
| `--result` | да | Результат или исход |
| `--memory-type` | нет | `decision` (по умолчанию), `pattern`, `bugfix`, `context`, `antipattern` |
| `--tags` | нет | Теги через запятую |
| `--project` | нет | Идентификатор проекта |

```bash
engram store \
  --context "Токены протухали молча" \
  --action "Добавил middleware валидации с логированием" \
  --result "Ошибки auth сразу видны в логах" \
  --memory-type bugfix \
  --tags "auth,middleware" \
  --project api-server
```

## engram search

Ищет релевантные записи через гибридный поиск (вектор + полнотекст).

```bash
engram search --query "auth middleware архитектура"
```

| Флаг | По умолчанию | Описание |
|------|-------------|----------|
| `--query` | — (обязательный) | Поисковый запрос |
| `--limit` | `10` | Максимум результатов |
| `--project` | — | Фильтр по проекту |

```bash
engram search --query "паттерны обработки ошибок" --limit 5 --project my-api
```

## engram judge

Оценивает качество записи. Результаты обучают Q-Learning роутер.

```bash
engram judge --memory-id "abc-123-def" --score 0.8
```

| Флаг | Обязательный | Описание |
|------|-------------|----------|
| `--memory-id` | да | UUID оцениваемой записи |
| `--query` | нет | Запрос, по которому найдена запись |
| `--score` | нет | Явная оценка от 0.0 до 1.0 |

## engram delete

Удаляет запись по ID. Удаляет саму запись и её элемент в HNSW-индексе. Безопасно по внешним ключам: строки feedback tracking для записи очищаются в той же транзакции. Если ID не существует, возвращается ошибка not found.

```bash
engram delete --id "abc-123-def"
```

| Флаг | Обязательный | Описание |
|------|-------------|----------|
| `--id` | да | UUID удаляемой записи |

## engram status

Показывает статус системы: количество записей, размер индекса, ожидающие оценки.

```bash
engram status
```

```bash
engram status --format text
```

## engram consolidate

Управление консолидацией записей. Четыре подкоманды:

### engram consolidate preview

Находит кандидатов для дедупликации без изменений.

```bash
engram consolidate preview
engram consolidate preview --stale-days 30 --min-score 0.5
```

### engram consolidate analyze

Анализирует кандидатов с помощью LLM и возвращает рекомендации.

```bash
engram consolidate analyze
```

### engram consolidate apply

Применяет рекомендации: merge, delete, archive.

```bash
engram consolidate apply
```

| Флаг | По умолчанию | Описание |
|------|-------------|----------|
| `--stale-days` | из конфигурации (90) | Порог устаревания (дни) |
| `--min-score` | из конфигурации (0.3) | Минимальный score |

### engram consolidate log

Выводит историю консолидации (журнал операций merge/delete/archive), новейшие первыми.

```bash
engram consolidate log
engram consolidate log --limit 20
```

| Флаг | По умолчанию | Описание |
|------|-------------|----------|
| `--limit` | 50 | Максимум записей |

## engram train

Управление самообучением через trainer.

### engram train generate

Запускает trainer для генерации insights из паттернов в памяти.

```bash
engram train generate
```

### engram train list

Показывает активные insights.

```bash
engram train list
```

### engram train delete

Удаляет insight по ID. Работает только для записей с типом `insight`.

```bash
engram train delete --id "insight-uuid"
```

## engram version

Показывает версию engram.

```bash
engram version
engram version --format text
```
