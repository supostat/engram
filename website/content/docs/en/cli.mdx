# CLI Commands

Engram CLI works standalone with SQLite and on-disk HNSW indexes. No running server required.

Global flags:

- `--config <path>` — path to configuration file
- `--format <json|text|jsonl>` — output format (default `json`)

## engram init

Initializes the project's `.engram/` directory in the current working directory (creating `engram.db`, a minimal commented-out project-local `engram.toml`, bundled `AGENT.md`, `.mcp.json`, and a `.engram/` entry in `.gitignore`). On first run also writes a default `~/.engram/engram.toml` if none exists, then prints MCP server setup instructions.

```bash
engram init
```

This command is handled before config loading and does not require an existing `engram.toml`.

## engram server

Starts a Unix socket server to accept JSON requests from the MCP server.

```bash
engram server
```

Resolves the socket path in this order: `ENGRAM_SOCKET_PATH` env override → `<project>/.engram/engram.sock` (when a `.engram/` marker is found by walking up from cwd, like `.git`) → `server.socket_path` from `engram.toml` (legacy fallback). The database is resolved analogously via `ENGRAM_DB_PATH` → `<project>/.engram/engram.db`. If the per-project DB is missing but a legacy `~/.engram/engram.db` exists, startup aborts with a hint to run `engram migrate`. Background reindexing runs at `reindex_interval_secs` interval.

## engram migrate

Imports memories from the legacy global database (`~/.engram/engram.db`) into the current project's `<project>/.engram/engram.db`. By default filters rows whose `project` field exactly matches the cwd basename; rows with a NULL `project` are skipped unless `--all` is passed.

```bash
cd /path/to/project
engram migrate --dry-run   # preview only — write nothing
engram migrate             # import rows tagged with current project
engram migrate --all       # also import NULL-project / mismatched rows
```

| Flag | Default | Description |
|------|---------|-------------|
| `--all` | off | Include rows whose `project` is NULL or differs from the cwd basename |
| `--dry-run` | off | Report what would be migrated without writing to the destination |

## engram reembed

Recomputes embeddings for every memory in the project database using the currently configured `embedding.provider` and `embedding.model`. Replaces vectors in HNSW (delete + insert per memory), writes fresh BLOBs to SQLite with `indexed=true`, clears the in-memory cache at the start, and records the active model in `schema_meta.embedding_model` once every memory succeeds.

```bash
engram reembed             # one-shot reembed under the active provider
engram reembed --force     # reserved for future safety thresholds (no-op today)
```

Use after changing `embedding.model` in `engram.toml`. The startup guard added in 0.3 refuses to boot the daemon with `[6020] EmbeddingModelMismatch` until `engram reembed` finishes cleanly.

Per-memory failure (provider unavailable, HNSW error) flips that memory's `indexed=0` so it can be retried later — the run reports `failed > 0` in its JSON summary and does **not** update `schema_meta`. Database write failures abort the run and propagate. The reembed command runs in the CLI path and intentionally bypasses the startup guard, so it can fix a mismatched database without the daemon refusing to start first.

| Flag | Default | Description |
|------|---------|-------------|
| `--force` | off | Reserved for future safety thresholds (no-op in 0.3.0) |

## engram store

Saves a new memory record.

```bash
engram store \
  --context "Auth module became monolithic, 800 lines" \
  --action "Split into middleware + handler + validator" \
  --result "Each file < 200 lines, tests isolated"
```

| Flag | Required | Description |
|------|----------|-------------|
| `--context` | yes | Situation or context |
| `--action` | yes | Action taken |
| `--result` | yes | Result or outcome |
| `--memory-type` | no | `decision` (default), `pattern`, `bugfix`, `context`, `antipattern` |
| `--tags` | no | Comma-separated tags |
| `--project` | no | Project identifier |

```bash
engram store \
  --context "Tokens expired silently" \
  --action "Added validation middleware with structured logging" \
  --result "Auth failures surface immediately with error codes" \
  --memory-type bugfix \
  --tags "auth,middleware" \
  --project api-server
```

## engram search

Searches for relevant records using hybrid search (vector + full-text).

```bash
engram search --query "auth middleware architecture"
```

| Flag | Default | Description |
|------|---------|-------------|
| `--query` | — (required) | Search query |
| `--limit` | `10` | Maximum results |
| `--project` | — | Filter by project |

```bash
engram search --query "error handling patterns" --limit 5 --project my-api
```

## engram judge

Rates a memory's quality. Results train the Q-Learning router.

```bash
engram judge --memory-id "abc-123-def" --score 0.8
```

| Flag | Required | Description |
|------|----------|-------------|
| `--memory-id` | yes | UUID of the memory to judge |
| `--query` | no | Query that found this memory |
| `--score` | no | Explicit score from 0.0 to 1.0 |

## engram status

Shows system status: memory count, index size, pending judgments.

```bash
engram status
```

```bash
engram status --format text
```

## engram consolidate

Manages memory consolidation. Three subcommands:

### engram consolidate preview

Finds deduplication candidates without making changes.

```bash
engram consolidate preview
engram consolidate preview --stale-days 30 --min-score 0.5
```

### engram consolidate analyze

Analyzes candidates with LLM and returns recommendations.

```bash
engram consolidate analyze
```

### engram consolidate apply

Applies recommendations: merge, delete, archive.

```bash
engram consolidate apply
```

| Flag | Default | Description |
|------|---------|-------------|
| `--stale-days` | from config (90) | Stale threshold (days) |
| `--min-score` | from config (0.3) | Minimum score |

## engram train

Manages self-learning through the trainer.

### engram train generate

Runs the trainer to generate insights from memory patterns.

```bash
engram train generate
```

### engram train list

Lists active insights.

```bash
engram train list
```

### engram train delete

Deletes an insight by ID. Only works for records with type `insight`.

```bash
engram train delete --id "insight-uuid"
```

## engram version

Shows the engram version.

```bash
engram version
engram version --format text
```
