# Configuration

Engram splits state across two locations:

- **Global** (`~/.engram/`) — `engram.toml` (base config layer, holds API keys and defaults), `models/` (ONNX artifacts shared across projects).
- **Per-project** (`<project>/.engram/`) — `engram.toml` (project-local config overrides), `engram.db` (SQLite) and `engram.sock` (Unix socket). Discovery walks up from cwd looking for `.engram/`, like `.git`.

## Layered config

`engram.toml` is resolved as **layers**, with priority `env > project-local > global`:

1. **Global** `~/.engram/engram.toml` — the base layer. Holds API keys and defaults. `embedding.api_key` and `llm.api_key` always come from here.
2. **Project-local** `<project>/.engram/engram.toml` — discovered by walking up from the current directory looking for a `.engram/` marker, like `.git`. When present, it is deep-merged field-by-field over the global layer: fields it sets override the global value, fields it omits inherit from global. Nested tables (`[embedding]`, `[hnsw]`, …) merge recursively. **An `api_key` set in a project-local file is ignored** — secrets always come from global.
3. **`ENGRAM_*` environment variables** — applied last, highest priority. They override values from either file layer.

The `--config <path>` CLI flag, when passed, replaces the global layer with the explicit file. If no config file exists at all, built-in defaults apply.

`engram init` writes a minimal, fully-commented project-local `engram.toml` template into `<project>/.engram/`. The template contains only non-secret override fields, all commented out — an untouched file inherits the global config verbatim.

## Path resolution

Database and socket paths are resolved at runtime, not from the config file:

| Resource | Priority order |
|----------|----------------|
| Database | `ENGRAM_DB_PATH` env → `<project>/.engram/engram.db` (per-project, found by walking up from cwd) → `~/.engram/memories.db` (legacy fallback) |
| Socket   | `ENGRAM_SOCKET_PATH` env → `<project>/.engram/engram.sock` (when a project `.engram/` exists) → `server.socket_path` from `engram.toml` (with `~` expanded) |

The `database.path` and `server.socket_path` values in `engram.toml` are kept as fallbacks for the case when no `.engram/` marker is found while walking up. `engram server` aborts on startup if a legacy `~/.engram/engram.db` exists but no per-project `.engram/engram.db` does — the error points at `engram migrate`.

## engram.toml structure

```toml
[database]
# Fallback only — runtime prefers per-project .engram/engram.db.
path = "~/.engram/memories.db"

[embedding]
provider = "voyage"         # voyage | ollama | deterministic
model = "voyage-4"
# host = "http://localhost:11434"   # ollama only; ENGRAM_OLLAMA_HOST overrides
# api_key = "..."           # or via ENGRAM_VOYAGE_API_KEY
# output_dimension = 1024   # optional; Voyage-4 Matryoshka (256/512/1024/2048). Must match [hnsw].dimension.
hyde_threshold = 0          # 0 = HyDE disabled (default); N>0 enables HyDE for queries shorter than N words

[llm]
provider = "openai"         # openai | ollama | local
model = "gpt-4o-mini"
# host = "http://localhost:11434"   # ollama only; ENGRAM_OLLAMA_HOST overrides
# api_key = "..."           # or via ENGRAM_OPENAI_API_KEY

[server]
# Fallback only — runtime prefers per-project .engram/engram.sock.
socket_path = "~/.engram/engram.sock"
reindex_interval_secs = 3600

[hnsw]
max_connections = 16
ef_construction = 200
ef_search = 40
dimension = 1024

[consolidation]
stale_days = 90
min_score = 0.3

[trainer]
trainer_binary = "engram-trainer"
trainer_timeout_secs = 300
models_path = "~/.engram/models"
```

## Sections

### database

| Parameter | Default | Description |
|-----------|---------|-------------|
| `path` | `~/.engram/memories.db` | Fallback SQLite path (overridden at runtime by per-project `.engram/engram.db` — see [Path resolution](#path-resolution)) |

### embedding

| Parameter | Default | Description |
|-----------|---------|-------------|
| `provider` | `voyage` | Embedding provider: `voyage`, `ollama`, or `deterministic` |
| `model` | `voyage-4` | Embedding model |
| `api_key` | — | API key (prefer environment variable) |
| `dimension` | `1024` | Embedding vector dimension |
| `output_dimension` | — | Optional Voyage-4 Matryoshka truncation (256/512/1024/2048). When set, must equal `[hnsw].dimension`. Omit to use the API default (1024). |
| `hyde_threshold` | `0` | HyDE opt-in: `0` disables HyDE; `N > 0` enables it for queries with fewer than `N` words |

> Changing `model` after a database has been populated triggers `[6020] EmbeddingModelMismatch` at the next `engram server` start. Run [`engram reembed`](/docs/cli#engram-reembed) once to recompute every vector with the active provider, then restart the daemon.

The `deterministic` provider requires no API key and generates deterministic embeddings from text. Suitable for testing.

The `ollama` provider talks to a local [Ollama](https://ollama.com) daemon (default host `http://localhost:11434`, overridable via `ENGRAM_OLLAMA_HOST`) and needs no API key — e.g. `model = "qwen3-embedding:0.6b"` at 1024-dim. Switching an existing database to it is a one-time [`engram reembed`](/docs/cli#engram-reembed).

When `hyde_threshold > 0`, short queries trigger an LLM call that produces a hypothetical memory, and the embedding is built from the hypothesis instead of the raw query. Trade-off: cache miss adds ~1.5s latency per query but improves recall on terse queries; cache is keyed by the **original query**, so repeated identical queries hit the cache without an extra LLM call.

### llm

| Parameter | Default | Description |
|-----------|---------|-------------|
| `provider` | `openai` | LLM provider: `openai`, `ollama`, or `local` |
| `model` | `gpt-4o-mini` | Text generation model |
| `api_key` | — | API key (prefer environment variable) |

The `local` provider uses an ONNX model from `trainer.models_path` for local generation without API calls.

The `ollama` provider runs LLM features (judge, HyDE, consolidation) on a local Ollama chat model (e.g. `model = "qwen3:4b"`) with no API key; if the daemon is unreachable, judge falls back to heuristics.

### server

| Parameter | Default | Description |
|-----------|---------|-------------|
| `socket_path` | `~/.engram/engram.sock` | Fallback Unix socket path (overridden at runtime by per-project `.engram/engram.sock` — see [Path resolution](#path-resolution)) |
| `reindex_interval_secs` | `3600` | Reindex interval (seconds, 0 = disabled) |

### hnsw

| Parameter | Default | Description |
|-----------|---------|-------------|
| `max_connections` | `16` | Maximum connections in HNSW graph |
| `ef_construction` | `200` | Accuracy during index construction |
| `ef_search` | `40` | Accuracy during search |
| `dimension` | `1024` | Vector dimension (must match embedding.dimension) |

### consolidation

| Parameter | Default | Description |
|-----------|---------|-------------|
| `stale_days` | `90` | Stale record threshold (days) |
| `min_score` | `0.3` | Minimum score for consolidation candidates |

### trainer

| Parameter | Default | Description |
|-----------|---------|-------------|
| `trainer_binary` | `engram-trainer` | Trainer binary path or name |
| `trainer_timeout_secs` | `300` | Trainer execution timeout (seconds) |
| `models_path` | `~/.engram/models` | Directory for ONNX models |

## Environment variables

| Variable | Overrides |
|----------|-----------|
| `ENGRAM_VOYAGE_API_KEY` | `embedding.api_key` (when provider=voyage) |
| `ENGRAM_OPENAI_API_KEY` | `llm.api_key` (when provider=openai) |
| `ENGRAM_OLLAMA_HOST` | `embedding.host` / `llm.host` (when provider=ollama) |
| `ENGRAM_DB_PATH` | `database.path` |
| `ENGRAM_SOCKET_PATH` | `server.socket_path` |
| `ENGRAM_EMBEDDING_MODEL` | `embedding.model` |
| `ENGRAM_LLM_MODEL` | `llm.model` |
| `ENGRAM_TRAINER_BINARY` | `trainer.trainer_binary` |
| `ENGRAM_TRAINER_TIMEOUT` | `trainer.trainer_timeout_secs` |
| `ENGRAM_MODELS_PATH` | `trainer.models_path` |

## Default paths

```
~/.engram/                       # global — shared across projects
  engram.toml                    # configuration (API keys, defaults)
  models/                        # ONNX models from trainer

<project>/.engram/               # per-project — created by `engram init`
  engram.toml                    # project-local config overrides (deep-merged over global)
  engram.db                      # SQLite database
  engram.sock                    # Unix socket server
  *.hnsw                         # serialized HNSW indexes
  AGENT.md                       # bundled agent instructions
```

## Initialization

The `engram init` command creates the project's `.engram/` directory in the current working directory (with `engram.db`, a minimal commented-out project-local `engram.toml`, the bundled `AGENT.md`, an `.mcp.json`, and a `.engram/` entry in `.gitignore`). On first run it also writes a default `~/.engram/engram.toml` if none exists. Existing files are never overwritten.

To import data from a previous global install (`~/.engram/engram.db`), see [`engram migrate`](/en/docs/cli#engram-migrate).
